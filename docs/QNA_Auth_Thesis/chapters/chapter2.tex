\section{Literature Review}

\begin{enumerate}

\item PRNU-Bench is an innovative create-a-benchmark and model for PRNU-based camera identification

arXiv Sept 2025 — https://arxiv.org/abs/2509.17581

\begin{itemize}
\item PRNU-Bench provides a standardised, large-scale benchmark for devices launched between 2021 – 2024.
\item PRNU-Bench supports generalisable and open-set camera identification using the Hadamard product similarity matrix.
\item PRNU-Bench leverages multiple images of natural scenes from each device, does not provide single-sample live authentication results.
\item The QNA-auth Tool was designed to allow single-sample live captures from each device per authentication session. The QNA-auth Tool directly addresses the multiple-image registration overhead issue that PRNU-Bench exposed.
\end{itemize}

\item CAMPrints makes use of the ``fingerprints'' of digital cameras

ACM MobiSys June 2025 — https://sunbangjie.github.io/files/CAMPrints.pdf

\begin{itemize}
\item CAMPrints extends PRNU modelling to build encoding representations that can withstand cropping, resizing and distortion.
\item CAMPrints enables matching between cameras after post-processing transformations have taken place.
\item To build encoding representations, CAMPrints required extensive datasets comprised from many different devices; the level of robustness to real-time compressing frames has not yet been fully validated.
\item QNA-Auth generates versioned features from noise residuals prior to being subjected to any type of compression pipeline, therefore QNA-Auth does not need to address the uncertainty caused by transformations related to the development of ``fingerprints'' through the methodolgy described in CAMPrints.
\end{itemize}

\item New Developments in Camera Identification and Linking Devices Through Deep Learning

\begin{itemize}
\item Recent literature reviews methods of identifying cameras via deep learning from 2016 through 2025, showing the superiority of encoder-decoder architectures to classical methods under both compression and brightness conditions. All reviewed camera identification methods are forensic and offline, without a provision for online authentication or replay attack.
\item The QNA-Auth algorithm uses audio noise characteristics to establish a similarity metric space to determine whether two devices (cameras) are the same or different and presents an online solution that bridges the gap between forensic identification methods described in the literature review and authentication methods.
\end{itemize}

\item Two Factor Authentication Using Acoustic Fingerprinting in the Modulation Domain (Ren et al.)

\begin{itemize}
\item Acoustic signals emitted from speaker and microphone pairs produce two factor authentication (2FA) fingerprints using magnitude and phase defects. The 2FA is capable of distance-independent operation to produce a robust acoustic fingerprint, i.e., does not vary between individuals regardless of distance between devices.
\item Active beeping from each enrolled device and the device attempting to login is required for two factor authentication; passive capture of audio from either device is not supported. To enable the use of QNA-Auth, passive fingerprinting of the microphone is possible to provide a similarity measure for all standard authentication methods without requiring a second active device.
\end{itemize}

\item Applying Unsupervised Contrastive Learning to Achieve Stronger RF Device Fingerprints Across Time Zones (Chen et al.)

arXiv / IEEE July 2022 — https://arxiv.org/abs/2403.04036

\begin{itemize}
\item Using a contrastive-based learning framework with matching sets of positive and negative signals can generate device embeddings that are invariant to the domain (e.g., the type of RF signal). When compared to traditional approaches using supervised learning, unsupervised contrastive-based learning improves accuracy by 10 to 28\% when tested with temporal domain shifts. This study only considered RF signals, and it will take additional work before the methodology can be applied cumulatively to account for noise from both a visual camera and analog microphone.
\item The QNA-Auth approach also utilizes the contrastive-based learning framework through a siamese-like architecture, implementing a physical measurement-based biometric for a two-factor authentication system.
\end{itemize}

\item Using Acoustic Fingerprinting to Provide a Robust Phone 2FA Authentication Mechanism (Ren et al.)

IEEE Transactions on Mobile Computing, Volume 23, No. 12 (2024) — https://ieeexplore.ieee.org/document/10505808

\begin{itemize}
\item Establishing a speaker/microphone-specific 2FA fingerprint is done by using a combination of acoustic fingerprinting measurement technologies to detect the time from when the beep signal is created to when it arrives at the receiver. The resulting fingerprint is reliable across large distances, however, it is heavily reliant on the establishment of precise timing of the beep signal and may be susceptible to replaying or spoofing audio recordings. The QNA-Auth system eliminates replay or unauthorized playback of the 2FA fingerprint through nonce binding, as each and every 2FA fingerprint recorded in a specific session must be replayed using a valid nonce.
\end{itemize}

\item Multimodal Biometric Identification: Utilizing CNN Architectures \& Fusion Techniques (Alshardan et al.)

PeerJ Computer Science, 10/2024 - https://doi.org/10.7717/peerj-cs.2440

\begin{itemize}
\item The use of weighted score-level fusion with two CNN extracted Bio-metric modalities produces consistently lower error rates over existing single-modality baselines.
\item The feature's quality is enhanced through a Contrast Limited Adaptive Histogram Equalization (CLAHE) pre-processing step prior to fusion.
\item The weights on the input (features derived from different modalities) do not change from fixed values after training has been completed for adaptive means of re-weighting during inference for poorly functioning modalities; although there is no means to re-weight other than using weights that have been predetermined.
\item QNA-Auth uses a method of static constitutive weightings (e.g., camera = 0.7 \& microphone = 0.3) on those data points contained within the three confidence bands for both modalities allowing for graceful degradation if either of them are weakly concurrent with their given confidence bound.
\end{itemize}

\item RFC 5869: HMAC based Extract-and-Expand Key Derivation Function (HKDF)

IETF, 2010 - https://datatracker.ietf.org/doc/html/rfc5869

\begin{itemize}
\item HKDF securely derives session-bound cryptographic keys from many entropy sources.
\item HKDF enables nonce-bound challenge/response-based protocols to render replayed credentials un-replayable due to the entropy-based ``nonce''.
\item HKDF assumes the source of entropy is of high-quality, stable input; where an entropy source such as; a noisy biometric template(s) requires special attention when normalizing to prevent entropy collapse.
\item QNA-Auth passes an embedded vector of quantized, versioned data to HKDF, ensuring consistency of RNG across HVK, while Session binding each session to a nonce issued to the server.
\end{itemize}

\item Poisoning Attacks Via Unsupervised Template Updating (Biometric) Backdoors

arXiv references through to 2024–2025; URL: https://arxiv.org/abs/1905.09162

\begin{itemize}
\item This paper demonstrates that adversarial input will result in the gradual biometrical update of the target's biometric template by poisoning unsupervised template update systems. The analysis of this attack is limited to unimodal (single) updates and does not cover multi-modal fusion of gated updates. The QNA-Auth implementation of the EMA-gated re-enrollment policy will only permit updates to templates where the new biometric samples scored above the strong accepted threshold and will block the low-confidence impostor sample from poisoning the template.
\end{itemize}

\item Hardware Security for IoT Identity Assurance

IEEE Communications Surveys \& Tutorials (2024); DOI: 10.1109/COMST.2024.3355168

\begin{itemize}
\item This paper reviews, describes, and evaluates the various hardware-rooted identity assurance mechanisms that are currently being deployed in IoT and mobile devices to protect user data (TEE, TPM, PUF, and secure elements) and then compares their features. The paper confirms that TEE-isolated key storage (for example, Apple's Secure Enclave) is the best identity assurance mechanism available at the application layer (i.e., after the operating system). Access to TEE-protected keys requires either vendor permission (e.g., user agreement) or privileges at the operating system-level; therefore, access to TEE-protected keys is not possible from browser-based applications or applications from third parties. The QNA-Auth architecture was specifically designed to manage HKDF keys according to the isolation patterns described in this paper, and the entire QNA-Auth architecture is TEE-upgradeable for enterprise-level deployments.
\end{itemize}

\end{enumerate}

\section{Research Gap}

The surveyed literature, while extensive, leaves several critical limitations unaddressed in combination:

\begin{itemize}
    \item \textbf{Real-time passive origin verification:} Camera and microphone fingerprinting methods reviewed in the literature have been developed for off-line forensic workflows. Previous systems exist to structurally verify passive, dark-frame camera noise, and ambient microphone noise as a real-time challenge/response web authentication method without the need for specialized hardware or explicit user credentials. 

    \item \textbf{Single-device Passive Acoustic Capture:} Current active signal emission-based methods of acoustic fingerprint authentication require active communications between two cooperating devices when enrolling and authenticating (logging in.) there appears to be no method, using a single microphone device for passive fingerprint generation and authentication as part of a routine web-based login flow in previous publications.

    \item \textbf{Compensating for sensor drift while minimizing the risk of poisoning:} The performance of consumer-grade sensors declines gradually over time resulting in an increasing mismatch between the enrolled templates and the current live samples. The biometric literature has some discussion regarding the periodic updating of templates, but as demonstrated in Mazumdar et al. Mazumdar2019Backdoors, there is a vulnerability of these rolling template updates (uncontrolled rolling template updates) to being poisoned due to a lack of gating controls. An acceptable method for re-enrollments (compensating for drift) without being compromised by adversarial manipulation has never been demonstrated in the context of hardware device fingerprinting. 

    \item \textbf{Calibrated multimodal score-level fusion with graceful degradation:} Methods utilizing a single sensor can yield a higher error rate than when multiple sensors are involved due to environmental interference. Therefore, using calibrated score-level fusion with modality-specific confidence bands (that will degrade gracefully when only one modality produces a bad signal) has not been attempted for hardware noise fingerprinting to provide authentication.

    \item \textbf{Nonce-bounded Replay Resilience of Sensor Based Credentials:} Prior to our work that derives cryptographic materials from biometric or sensor based features shown that the responses produced do not bind the derived response to per session server nonce thereby placing the channel subject to replay of prior observations of the embedded values.
\end{itemize}

\section{Objectives}

The primary objectives of this project are:

\begin{itemize}
    \item Construct a pipeline to passively capture image and sound from a consumer camera and microphone, respectively, that will 1) capture 50 dark frames at 60 Hz from the camera using OpenCV and 50 ambient sound samples at 44,100 Hz from the microphone, and 2) create 50-dimensional canonical feature vectors based on statistical moments, spectral features, Shannon entropies, and autocorrelations.\cite{shannon1948mathematical, welch1967use}.

    \item Develop a Siamese neural network architecture for the two modalities; both share a four-layer embedding network structure (input: 50-dim; hidden1: 256; hidden2: 256; output: L2 normalized 128-dim) using a triplet loss function with a margin of 1.0 to differentiate between same and different devices based on cosine similarity. \cite{bromley1993signature, schroff2015facenet}.

    \item Create a non-binary decision-making engine that takes concatenated cosine similarity scores from both modalities, assigns calibrated weights to each modality (camera = 0.85, microphone = 0.15) and classifies the final output into three bands of confidence: strong (0.971), uncertain (0.9192-0.916), reject (0.9190).

    \item The server generates a 32-bit nonce, which remains valid for a period of 60 seconds. The client calculates its session MAC via HKDF-SHA256 on the quantized embedding of the device it will authenticate from, along with the original nonce provided by the server. The server checks the validity of both the nonce and session MAC by performing a constant-time process to compare their HMAC outputs.\cite{krawczyk2010hkdf}.

    \item To implement a process of re-enrolling onto the device using an EMA algorithm (alpha = 0.2), this mechanism requires a minimum of two consecutive strong-accept matches before allowing any modification to the bank per-device template (with up to four per-modality) to gracefully accommodate minor variances in hardware (e.g., gradual camera drift) while still maintaining a high confidence in detecting attacks (low-confidence impostors).\cite{mazumdar2019backdoors}.
\end{itemize}

\section{Problem Statement}

The fundamentals of establishing that an interaction with a digital asset has taken place within a trust framework is dependent on the currently accepted methods of user authentication. The most common forms of establishing identity are traditional, functionally-based credentials (password, pin, etc.) and possession-based (hardware token, etc.). Both of these cues do not provide any evidence that the device being used to authenticate is the same device that was used to enroll the user. Therefore, both methods can be compromised allowing unauthorized access through credential theft, session hijacking and device emulation. The majority of modern consumer-grade devices contain unique electronic characteristics based on their manufacturing tolerances that will remain constant while recording pictures or audio over time and cannot be created with software emulation. Therefore, there is a need for a passive authentication layer of software that will interactively capture these electronic characteristics at the time of login, compare them to the characteristics of the user that has been previously enrolled using some similarity measure, and associate the results with a cryptographic challenge that only applies to one session, therefore allowing for the continuous verification of the authenticity of the device being used without requiring active participation on the part of the user or the need for any additional hardware not already found on standard computer systems

\section{Project Plan}

The development of the QNA-Auth prototype is structured across five sequential phases:

\begin{itemize}
    \item \textbf{Phase 1 - Data Collection and Preprocessing:} Collecting dark frame camera noise and ambient microphone noise datasets from various consumer electronics. Using a well-known method to extract 50-dimensional feature vectors (using statistical, spectral, entropy, and auto-correlation data) and creating labeled triplet sets for the purpose of training models. 

    \item \textbf{Phase 2 - Model Architecture and Training:} Creating and training Siamese embedding networks separately on the camera and microphone modalities using triplet Loss (with a margin of 1.0). Training models in an iterative fashion over 50 epochs using a batch size of 32, and a learning rate of 10 ^ 3 and validating each epoch to save the best performing model.

    \item \textbf{Phase 3 - Core Authentication Logic:} Implementing a weighted fusion engine along with three-band confidence thresholding and a minimum identification margin check requiring that the claimed device score 0.02 or higher than the next highest enrolled candidate. 

    \item \textbf{Phase 4 - Security and Drift Adaptation:} Integrating the HKDF nonce-bound challenge-response layer providing a challenge expiration of 60 seconds, along with an anti-replay tracking period of 300 seconds for the responses and allowing for the implementation of the EMA-gated template re-enrolment mechanism using alpha = 0.2 with a minimum of two consecutive strong accept gate matches.

    \item \textbf{Phase 5 — Prototype Integration and Evaluation:P/} Deployment of the complete pipeline using FastAPI for the back-end (the SQLite audit log will be created through SQLAlchemy) and React 18 / TypeScript / Vite for the front-end, with endpoint finalization  (\texttt{/enroll}, \texttt{/challenge}, \texttt{/verify}, \texttt{/authenticate}), and actual measurements of the false acceptance rate, false rejection rate, and total authentication latency.
\end{itemize}
