\section{Literature Review}


\subsection{Large-Scale Test of Sensor Fingerprint Camera Identification}
This paper evaluates PRNU-based camera fingerprinting across large datasets and demonstrates its scalability in real-world scenarios. It shows that sensor pattern noise remains stable across images from the same device. The study validates PRNU as a reliable identifier for digital cameras. 
\vspace{1em}

\subsection{Determining Image Origin Using Sensor Noise}
This work introduces a maximum-likelihood approach for estimating PRNU noise patterns. It improves identification accuracy compared to traditional correlation-based methods. The paper provides a strong statistical basis for camera source attribution. 
\vspace{1em}

\subsection{Digital Camera Identification from Sensor Pattern Noise}
This seminal work establishes PRNU as a unique fingerprint for digital cameras. It shows that manufacturing imperfections create consistent sensor noise patterns. The method enables reliable mapping of images to source devices. 
\vspace{1em}

\subsection{Do You Hear What I Hear? Fingerprinting Smart Devices through Acoustic Components}
This paper explores acoustic fingerprinting using microphones and speakers. It demonstrates that hardware imperfections introduce unique audio distortions. These characteristics can be used for passive device identification. 
\vspace{1em}

\subsection{Siamese Neural Networks for One-Shot Image Recognition}
This work introduces Siamese networks for learning similarity between inputs. It enables classification using minimal training samples through embedding comparison. The model generalizes well to unseen classes in verification tasks. 
\vspace{1em}

\subsection{FaceNet: A Unified Embedding for Face Recognition and Clustering}
FaceNet proposes a triplet-loss-based embedding for face recognition. It maps inputs into a feature space where distances reflect identity similarity. The approach achieves high accuracy and scalability in large datasets. 
\vspace{1em}

\subsection{Handbook of Multibiometrics}
This book presents comprehensive techniques for multi-modal biometric systems. It discusses fusion strategies at feature, score, and decision levels. The work shows how combining modalities improves system reliability. 
\vspace{1em}

\subsection{RFC 5869: HMAC-based Extract-and-Expand Key Derivation Function (HKDF)}
This standard defines HKDF for deriving secure cryptographic keys. It extracts and expands entropy from input sources into usable keys. HKDF is widely used in modern secure communication systems. 
\vspace{1em}

\subsection{Template Update Methods in Adaptive Biometric Systems}
This paper studies biometric template drift over time. It reviews methods for updating stored biometric representations. The work emphasizes adaptive mechanisms to maintain accuracy. 
\vspace{1em}

\subsection{Demystifying ARM TrustZone: A Comprehensive Survey}
This survey explains the architecture and security model of ARM TrustZone. It highlights trusted execution environments for secure key storage. The work provides insights into hardware-backed authentication systems. 

\section{Research Gap}
While existing research covers hardware forensics and deep learning extensively, several critical gaps remain:
\begin{itemize}
	\item \textbf{What current systems fail to address:} Existing techniques struggle to verify the physical origin of a digital session securely in real-time without falling back to specialized cryptographic hardware or easily spoofed passwords.
	\item \textbf{Limitations in previous studies:} Most sensory forensics work focuses on offline analysis of images or audio for evidence, rather than structuring it as a real-time web-authentication process.
	\item \textbf{Missing features:} Continuous, rolling verification that dynamically adapts to a hardware's gradual physical wear (sensor drift) over time is largely absent from current fingerprinting implementations.
	\item \textbf{Performance weaknesses:} Single-sensor approaches (relying only on a camera or only on a microphone) often suffer from higher False Acceptance Rates or struggle under environmental interference.
	\item \textbf{Scalability and usability issues:} Hardware tokens lack the scalability required for rapid consumer deployment, whereas the proposed software-only solution leverages ubiquitous, pre-existing components.
\end{itemize}

\section{Objectives}
The primary objectives of this project form the basis for the QNA-Auth system:
\begin{itemize}
	\item To design a mechanism that securely captures and processes ambient noise from consumer cameras and microphones into high-dimensional, biometric-like embeddings.
	\item To develop and train a Siamese neural network capable of computing similarity scores to differentiate between identical and distinct sensor hardware.
	\item To construct a non-binary decision engine utilizing a weighted fusion model (70 percent camera, 30 percent microphone) and defined confidence bands for flexible authentication.
	\item To implement a secure challenge-response authorization flow that uses nonce-bound Hash-based Message Authentication Codes to protect the embeddings against replay attacks.
	\item To deploy a rolling Exponential Moving Average update mechanism that safely incorporates new sensor templates over time to accommodate hardware drift.
\end{itemize}

\section{Problem statement}
Establishing definitive trust in the physical origin of a digital session is overwhelmingly dependent on static, explicit credentials or highly specialized hardware. Consequently, digital systems remain vulnerable to session hijacking, credential theft, and advanced emulation. There is a critical need for a passive, software-evaluating authentication layer that leverages universal consumer hardware—specifically cameras and microphones—to continuously verify a device's unique physical manufacturing imperfections without requiring active user involvement.

\section{Project plan}
The development and validation of the QNA-Auth prototype are structured across the following primary phases:
\begin{itemize}
	\item \textbf{Phase 1: Data Collection and Preprocessing:} Gathering and sanitizing robust noise datasets from varied camera and microphone sensors to act as the foundational training data.
	\item \textbf{Phase 2: Model Architecture and Training:} Designing the deep learning feature extractors and the Siamese network topology, followed by iterative training to optimize the separation distance between distinct devices.
	\item \textbf{Phase 3: Core Algorithm Construction:} Developing the authentication logic, implementing the confidence band thresholding (strong accept, uncertain, reject), and calculating the sensor-weighted fusion results.
	\item \textbf{Phase 4: Security and Drift Adaptation:} Integrating the Exponential Moving Average logic for template re-enrollment and constructing the challenge-response API bound by server-side secrets.
	\item \textbf{Phase 5: Prototype Development and Evaluation:} Wrapping the models within the FastAPI backend architecture and Vite frontend, finalizing API routes, and running empirical evaluations on false acceptance and rejection parameters.
\end{itemize}
