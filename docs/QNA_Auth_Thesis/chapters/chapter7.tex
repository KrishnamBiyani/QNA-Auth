\section{Results Overview}

In this section of the report we provide an analysis of various aspects of the QNA-Auth system. We will evaluate how well it classifies data, embeds that data into the user’s session and how many users are able to pass through the defined thresholds. This analysis will include some qualitative information to describe the performance of the QNA-Auth system from a real world perspective.

\section{Classification Performance}

\begin{figure}[H]
\centering
\includegraphics[width=0.75\linewidth]{images/camera_confusion_matrix.png}
\caption{Confusion matrix at deployed threshold}
\label{fig:conf_matrix}
\end{figure}

As can be observed from the confusion matrix, the ability of the system to identify both real and fake/proxy samples is very good. The very high number of true positives (552) and true negatives (844) indicate that the system has a very strong discrimination ability.

However, the false negatives (318) are also high, indicating that the system has a large number of legitimate user transactions/activities that it did not allow through because of the high decision threshold used to prioritize security (i.e., to remove as many false acceptances as possible) over usability.

\section{Overall Discussion}

The findings indicate that the use of QNA-Auth allows for capturing device-specific information from sensor noise and can provide a reasonably effective means for differentiating between real and false samples.

The development of a highly effective 「impostor rejection」 system (i.e., low rate of false alarm), coupled with an increased rate of false alarm, suggests that the system may be sensitive to both the threshold used for making decisions and the variability caused by different environments. This also illustrates the ongoing tension between security and usability.

The relative performance of the authentication system was highly variable when using different input modalities. For example, fingerprinting using a camera was demonstrated to be a more reliable form of input than recording via a microphone.

The projected results increase the argument for the viability of the project by not altering the legal basis upon which a court could find the authentication system to be reliable. The experiments suggest that, as the dataset grows and extensive testing under stronger divisions of fused testing are undertaken, the authentication system has the potential to develop from a prototype with promise to a more competitive pipeline for authentication of short samples. This positive conclusion is of sufficient benefit to academia to continue with the project even with the results to-date being limited in quantity.

\section{Conclusion of Results}

The validity of sensor-based device authentication has been proved in practice, but not to the extent of forensic methods in terms of accuracy. Nevertheless, through the combination of machine learning, signal processing and security-focused system design, progression towards practical application of sensor-based authentication will be achieved.
