\section{System Architecture}

\subsection{Architecture}

The project follows a layered architecture where data collection, preprocessing, embedding, application logic, persistence, and user interface are separated into distinct layers. This ensures modularity, scalability, and ease of maintenance.

At startup, the backend initializes the model, preprocessing pipeline, and database connections to ensure efficient request handling.

\subsection{System Design}

\begin{figure}[H]
	\centering
	\includegraphics[width=0.95\linewidth]{images/system_design.pngg}
	\caption{System design pipeline of QNA-Auth}
	\label{fig:system_design}
\end{figure}

The system design follows a structured pipeline consisting of data capture, feature engineering, model processing, and decision logic. Sensor inputs from the camera and microphone are transformed into statistical and frequency-based features before being normalized. These features are passed through a Siamese network to generate embeddings and compute similarity scores. The final decision is made using weighted fusion and thresholding, ensuring robust and reliable device authentication.

The class diagram represents the modular organization of the system and the interaction between core components. The FastAPI server acts as the entry point, coordinating enrollment and authentication workflows. Classes such as DeviceEnroller, DeviceAuthenticator, and NoisePreprocessor handle data processing, feature extraction, and verification logic. The design ensures separation of concerns, making the system scalable, maintainable, and easy to extend.

\subsection{API Design}

The backend exposes a compact set of endpoints aligned with the system workflow:

\begin{itemize}
	\item \texttt{POST /enroll}
	\item \texttt{POST /authenticate}
	\item \texttt{POST /challenge}
	\item \texttt{POST /verify}
	\item \texttt{GET /devices}
	\item \texttt{GET /devices/\{device\_id\}}
	\item \texttt{DELETE /devices/\{device\_id\}}
	\item \texttt{GET /health}
	\item \texttt{GET /stats}
\end{itemize}

This API structure supports enrollment, authentication, device management, and system monitoring while maintaining simplicity and clarity.
